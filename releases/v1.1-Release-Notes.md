# Release Notes — v1.1 (Post-CR Improvements)

**Project:** Mini / Campus Event Registration System  
**Release Tag:** v1.1  
**Release Date:** 2025-12-29  
**Baseline Reference:** BL2 + post-BL2 improvements (approved CRs)

## Summary
Improves usability and registration management after the initial working release.

## Changes Since v1.0
- Enhanced “My Registered Events”:
  - Improved list + unregister support
  - Registration rules enforced (no duplicates, no self-registration)
- Navigation:
  - Shows registered-events count beside “My Registered Events”
- UI refinements:
  - Event list card styling and event details modal usability improvements
- Event editing:
  - Owner can update event image/description from the UI

## Compatibility
- Uses the same `localStorage` keys as v1.0.
- Existing users/events should remain compatible.

## Known Limitations
- Client-side only; no multi-device sync.
- Manual testing only.

## Verification (Manual Smoke Checklist)
- All v1.0 checks pass
- Unregister removes event from “My Registered Events”
- Count in nav updates immediately after register/unregister
- Owner edit actions are available only for owned events
